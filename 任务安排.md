# 任务安排

**Day 1：项目骨架 + 环境**

- 建 Spring Boot 项目
- 接入 MySQL / Redis / MongoDB / RabbitMQ / Elasticsearch
- 搭好多模块结构
- 配置基础实体、异常、统一返回
- 跑通健康检查和基础连接

产出目标：
项目能启动，五个中间件都能连通。

**Day 2：工单主流程**

- 完成工单创建、查询详情、状态流转
- MySQL 存主表，MongoDB 存详情
- 先手工分派也可以，保证主链路跑通

产出目标：
能完整演示“提单 -> 查看 -> 处理 -> 关闭”。

**Day 3：RabbitMQ + 自动分派**

- 建单后发 ticket.created
- 消费消息执行规则分派
- 规则先做简单版：分类 + 部门 + 关键字
- 分派后更新处理人待办

产出目标：
工单创建后能自动分到对应组或人。

**Day 4：Elasticsearch 搜索 + 知识库**

- 建工单索引和知识库索引
- 做工单搜索、知识搜索
- 关闭工单后一键沉淀知识
- 建单前支持相似问题推荐

产出目标：
有搜索体验，项目一下就不像 CRUD 了。

**Day 5：Redis 缓存 + SLA**

- 做待办数缓存、处理人负载缓存
- 做热点知识缓存
- 用 RabbitMQ 延迟/TTL 方案做 SLA 检查
- 超时后写告警日志或通知记录

产出目标：
系统有实时感和异步治理能力。

**Day 6：管理端基础 + 联调测试**

- 做简单管理接口：规则管理、SLA 管理、统计接口
- 测主流程和异常分支
- 补日志、幂等、防重复消费

产出目标：
从业务到中间件整条链路稳定。

**Day 7：收尾与项目包装**

- 补 README、架构图、流程图
- 准备演示数据
- 补 2-3 个核心测试
- 优化接口命名和代码结构
