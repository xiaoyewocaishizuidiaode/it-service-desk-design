# 模块

1. ticket-core
   负责工单领域能力：提单、分派、转派、接单、处理、关闭、评价。
2. knowledge-core
   负责知识库：知识文章、分类、标签、引用历史工单生成知识条目。
3. search-core
   负责统一搜索：工单搜索、知识库搜索、相似工单推荐、搜索建议。
4. rule-engine
   负责自动分派和 SLA 规则：按分类、部门、关键字、优先级匹配处理组。
5. message-center
   负责 RabbitMQ 事件发布和消费：工单创建、状态变更、通知、超时提醒。
6. cache-center
   负责 Redis 缓存：热点知识、待办数、工单统计、幂等和限流。
7. admin-report
   负责统计分析：处理时长、超时率、知识命中率、工单分类分布。
8. common-infra
   放公共能力：异常、鉴权、审计日志、统一响应、枚举、工具类、配置。

# 技术分配

- MySQL
  放强事务核心数据：用户、工单主表、工单流转记录、处理组、SLA 规则。
- MongoDB
  放非结构化和长文本：工单对话记录、操作详情、附件元数据、知识正文、审计快照。
- Elasticsearch
  放搜索索引：工单摘要索引、知识库索引、相似问题检索。
- Redis
  放热点缓存和实时状态：待办计数、排行榜、搜索建议缓存、幂等 token、分布式锁。
- RabbitMQ
  放异步事件：工单创建、自动分派、通知、SLA 延迟检查、索引更新。

# 项目流程

1. 员工提交工单
   填写标题、描述、分类、紧急程度、所属部门、联系人、附件。
2. 系统预检查
   先去 ES 检索相似工单和知识库文章，给用户推荐“可能已有答案”；如果用户仍提交，再进入正式建单。
3. 建单与自动分派
   工单主数据写入 MySQL，详情和上下文写入 MongoDB，然后发送 ticket_created 消息到 RabbitMQ。
   消费者根据分类、关键字、部门、优先级匹配处理组和处理人。
4. 工单处理流转
   支持 待分派 -> 待接单 -> 处理中 -> 待确认 -> 已关闭，中间允许转派、加急、挂起、补充说明。
5. SLA 检查与升级
   建单后根据优先级生成 SLA 检查任务，比如“30 分钟未接单”“4 小时未解决”。
   到期后由 RabbitMQ 延迟消息或 TTL + 死信队列触发检查；若超时，则自动告警、催办、升级给上级组。
6. 知识沉淀
   工单关闭后，处理人可以一键把解决方案沉淀成知识库文章，供后续搜索和推荐复用。
